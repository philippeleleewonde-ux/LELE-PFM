Code Problématique - calculateIndicatorData()

function calculateIndicatorData(kpiType, entries, pprSettings) { // ⚠️ PROBLÈME: Utilise la valeur PPR HEBDOMADAIRE simple const pprKey = `ppr_${kpiType.toLowerCase()}_weekly`; const objectif = pprSettings?.[pprKey] || 0; // = 615¥ !! // Économies = objectif - frais (formule SIMPLIFIÉE) const economiesRealisees = Math.max(0, objectif - totalFrais); return { objectif, // 615¥ au lieu de 1,230,486¥ economiesRealisees, // ~579¥ au lieu de 579,052¥ prevPrime: objectif * 0.33, prevTreso: objectif * 0.67, // ... }; }

Explication: Cette fonction de fallback utilise pprSettings.ppr_abs_weekly qui contient la valeur PPR hebdomadaire brute (~615¥), pas les valeurs calculées par le Récapitulatif qui agrègent N1+N2, calculent les scores de contribution, etc.

Bug Découvert - Double Appel setPerformanceData()

✅ CORRECT Ligne 2858

setPerformanceData( performanceIndicators, performanceTotals, businessLinePerformancesData, employeePerformancesData // ← AVEC employees );

❌ BUG Ligne 6500
